// How it works

Three commands. Total isolation.

1

You run mrc ~/your/repo

The launcher builds a Docker image, starts clipboard & notification proxies, finds your .sandboxignore files, and fires up the container with your repo mounted.

2

The firewall jams their radar

iptables locks the container down. Only api.anthropic.com, npm, and sentry can be reached. Your Postgres, Redis, AWS RDS, and everything else: blocked. IPv6: blocked. Host services: blocked.

3

Claude Code runs with full permissions

--dangerously-skip-permissions is safe here because the container IS the sandbox. Claude can do whatever it wants — inside the box.

// Features

All bleeps, sweeps, and creeps accounted for

🛡

iptables Firewall

Default DROP policy. Whitelisted domains only. IPv6 blocked. Host services blocked. Your databases are unreachable.

📋

Clipboard Proxy

Paste images and text into the container via a TCP proxy. Works through host.docker.internal. Supports PNG, TIFF, JPEG.

🔔

Desktop Notifications

Native macOS/Linux notifications when Claude finishes responding. Shows which repo and a summary. Configurable sound.

💾

Session Management

Auto-resume, named sessions, interactive picker with mrc pick. AI-generated names and summaries. Tool-miss detection on exit.

🕶

.sandboxignore

Recursive, like .gitignore. Place anywhere in the tree. Files appear empty, directories appear as empty dirs.

⚙

.mrcrc Config

Global defaults in ~/.mrcrc, per-repo overrides in <repo>/.mrcrc. One flag per line. CLI always wins.

📦

Multi-Instance

Run multiple sessions against the same repo. Each gets its own config volume and dynamic proxy ports. No collisions.

📊

Context Status Line

Live progress bar showing context window usage, 5h/7d rate limits, and session name. Color-coded warnings as you approach compaction.

// CLI Reference

mrc [options] [path] [-- args...]

FlagDescription
Options
-r, --rebuildForce a full image rebuild (no cache)
-v, --verboseShow Docker output
-n, --new [name]Start a new conversation (optionally named)
-w, --webAllow outbound HTTPS to any host
--no-summarySkip AI session summary on exit
--no-notifyDisable desktop notifications entirely
--no-soundDisable notification sound (still shows notification)
Commands
mrc statusShow active containers across repos
mrc pick [path]Interactive session picker (arrow keys)
mrc sessions lsList saved sessions
mrc sessions nameName a session
mrc sessions resumeResume a specific session
Environment
ANTHROPIC_API_KEYAPI key (also loaded from .env or ~/.config/mrc/.env)
MRC_PORT_BASEStarting port for proxy allocation (default: 7722)
MRC_HOMEOverride Docker context directory (advanced)

// Changelog

What's new

v1.0.0 — Ludicrous Speed

  • Standalone binary — Single-file install via curl | bash. No Node.js required on the host.
  • Session picker — mrc pick launches an interactive TUI to browse, select, or start sessions.
  • Context status line — Live progress bar showing context window usage, 5h/7d rate limits, and session name.
  • Auto-naming — Sessions are automatically named via Haiku API after enough conversation builds up.
  • Node.js rewrite — Host launcher and container scripts rewritten from bash/Python to Node.js for easier maintenance.

Session Management

  • Auto-resume — Re-opening a repo continues the last conversation automatically.
  • Named sessions — mrc --new bugfix starts a fresh conversation with a name.
  • Session subcommands — mrc sessions ls, name, resume.
  • Session summaries — AI-generated summaries saved on exit (requires API key).
  • Tool-miss detection — Reports missing tools with issue links on exit.

Desktop Notifications

  • Response-complete notifications — Native macOS notification when Claude finishes, with repo name and summary.
  • --no-notify — Disable notifications entirely.
  • --no-sound — Keep the notification, kill the Glass sound.

Security & Multi-Instance

  • Host network lockdown — Firewall only opens proxy ports. Your local Postgres, Redis, and everything else on the host are blocked.
  • Concurrent sessions — Multiple instances per repo, each with its own config volume and fresh session.
  • Dynamic port allocation — Each instance gets free ports starting from MRC_PORT_BASE.

Config & Ergonomics

  • .mrcrc config files — Global (~/.mrcrc) and per-repo defaults.
  • Recursive .sandboxignore — Place anywhere in the tree, entries resolve relative to their directory.
  • mrc status — See all running containers with uptime and flags.
